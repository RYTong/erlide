What is tracing?

"In software engineering, tracing is a specialized use of logging to record information about a program's execution. This information is typically used by programmers for debugging purposes, and additionally, depending on the type and detail of information contained in a trace log, by experienced system administrators or technical support personnel to diagnose common problems with software."
http://en.wikipedia.org/wiki/Tracing_(software)

Feature description

This feature uses an improved version of ttb (Trace Tool Builder, http://www.erlang.org/doc/man/ttb.html) for tracing purposes and provides grafical interface to it offering possibility of tracing Erlang programs directly from erlide.


Tracing control panel

Tracing is a specific kind of event logging - it logs "low level" information. Because of that user quickly gets overwhelmed with logged data and finds it rather difficult to browse it. This plugin also profides an user friedly way of browsing trace results - they are displayed in a form of a tree where each event is represented by different type of icon. Tree nodes can be unfolded to get more details about particular event.

Features

  • New eclipse perspective and views
  • Setting process flags
  • Tracing multiple nodes at the same time (also nodes started outside erlide)
  • Setting net nick time and cookie per node
  • Selecting processes which should be traced
  • Setting trace patterns for functions (support for match specifications)
  • New positions in outline's context menu for creating trace patterns from selected function
  • Saving and restoring trace patterns configuration
  • Saving and restoring nodes configuration
  • Browsing trace results in tree
  • Loading trace results from disk
